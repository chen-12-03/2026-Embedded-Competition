# 智慧工业边缘网关：物理异常注入与闭环控制系统整合报告

## 1. 物理异常沙盘架构设计 (Hardware Sandbox)
为了在答辩的 5~7 分钟内稳定复现工业现场的复杂故障，系统摒弃了随机破坏，采用变量绝对可控的机电与热力学微缩沙盘。

### 1.1 振动与过流复合靶标（机电域）
* **硬件拓扑**：5V 直流减速电机固定于底座，转轴预留加装偏心轮（如橡皮擦）的空间。供电正极串联 INA219 电流传感器与 5V 继电器模块。MPU6050 振动传感器刚性贴合于电机外壳。
* **注入逻辑**：
  * **高频微震（早衰模拟）**：加装偏心轮后，电机空载运行将产生周期性的机械不平衡。此状态下电流正常，但 MPU6050 可捕获极其微小的高频振动特征，作为 AI 模型的绝佳输入靶标。
  * **极限过载（故障模拟）**：向转轴施加急剧的物理摩擦（如海绵按压）引发电机堵转，触发 INA219 读数在毫秒级指数飙升。

### 1.2 瞬态热传导靶标（热力学域）
* **硬件拓扑**：SHT30 探头悬空对准 5V PTC 恒温发热板。发热板由独立开关或主板 GPIO 控制。
* **注入逻辑**：通电后发热板在 3~5 秒内迅速升温至 50℃，人为制造陡峭的温度梯度变化，用于触发平滑且迅速的高温告警曲线。

## 2. 软硬协同的闭环数据流 (Closed-Loop System)
物理异常产生后，需依托 ELF-RV1126B 平台，完成“感知 $\rightarrow$ 清洗 $\rightarrow$ 决策 $\rightarrow$ 执行”的完整系统响应。


### 2.1 连续时间信号采集与数字清洗
* **高频并发**：在 Linux 系统下部署多线程服务，通过 I2C 总线以 100ms~1000ms 周期高频轮询多路传感器状态。
* **去噪平滑**：引入线性时不变系统（LTI）的设计思维，在数据入队前套用滑动窗口移动平均，有效抑制 I2C 总线上的随机电磁毛刺，确保输入数据的信噪比。

### 2.2 时频特征提取与建模
* **时域特征**：提取清洗后数据的关键表征量，如计算窗口内的均方根（RMS）：
  $$X_{RMS} = \sqrt{\frac{1}{N}\sum_{n=1}^{N}x[n]^2}$$
* **频域特征**：运用傅里叶级数原理，将连续采集的振动时序信号转换为频域能量图谱，精准捕捉因机械早衰引起的高频谐波突变。

### 2.3 双轨融合判定引擎
采用“并联”逻辑兼顾系统的稳定性与前瞻性：
* **红警规则通道**：旁路复杂运算，直接对比绝对安全边界（如突变电流）。一旦越限，极速输出 Level 1 最高告警。
* **黄警 AI 通道**：将时频多维特征输入端侧 Isolation Forest 异常检测模型。若数据在特征空间中出现分布漂移，输出 Level 2 早期预警。

### 2.4 物理级反馈执行
* **声光与 UI 映射**：跨进程驱动本地 HMI（Qt/LVGL/Web）大屏界面泛红、弹窗；底层 GPIO 驱动有源蜂鸣器与警示灯爆发强感官刺激。
* **硬核急停闭环**：捕捉到红警瞬间，系统拉高 5V 继电器的控制引脚，物理切断测试电机的供电回路。源头切断，系统回归稳态。日志系统同步将异常时间、峰值落盘至 SQLite 或 CSV 以供复盘。

## 3. 现场全场景演示剧本 (Live Demo)
这一演示流程旨在将复杂的数学建模与底层时序控制具象化，赋予评委极强的视觉冲击力。

| 演示阶段 | 现场物理操作 | 底层系统与模型状态 | HMI大屏与外设表现 |
| :--- | :--- | :--- | :--- |
| **1. 稳态基线** | 电机平稳空转。 | I2C 高频采样，滤波器平滑处理。AI 异常分数极低。 | 绿灯常亮。HMI 曲线平滑，状态显示“运行良好”。 |
| **2. 亚健康注入** | 转轴加装偏心轮（轻微不平衡）。 | 规则通道静默。时频特征发生漂移，Isolation Forest 模型判定数据偏离常规分布。 | **黄警触发**。HMI 异常分数攀升，UI 提示“早期退化”。黄灯常亮/慢闪。 |
| **3. 极限故障** | 手动捏住转轴施加阻力（电机堵转）。 | INA219 电流瞬态飙升。规则引擎在毫秒级接管最高判定权，下达急停指令。 | **红警爆发**。红灯爆闪、蜂鸣器长鸣。继电器瞬间切断电机电源，电机停转。 |
| **4. 数据复盘** | 停止操作，展示终端。 | 系统自恢复，日志记录结束并封存入库。 | 展示日志详情与“规则 VS AI 预警时间对比”，量化算法价值。 |
